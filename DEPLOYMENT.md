# Deploying to a public, shareable URL

Stack: **Neon** (free permanent Postgres) + **Render** (free FastAPI backend) + **Streamlit Community Cloud** (free frontend). No server to maintain, nothing for visitors to install — they just open a link.

Account creation has to happen in your own browser (I can't sign up for services or enter credentials on your behalf). Everything else — repo prep, config files, schema — is already done. Steps below are in order.

## 1. Push the code to GitHub

```bash
cd "/Users/vasudhalohia/Desktop/Financial Data/indian-fin-platform"
git add -A
git commit -m "Initial commit: simplified 3-service stack, ready for cloud deploy"
```

Then create a new repo at github.com/new (public or private, doesn't matter), and run the two commands it shows you, e.g.:

```bash
git remote add origin https://github.com/<your-username>/indian-fin-platform.git
git branch -M main
git push -u origin main
```

## 2. Database — Neon

1. Sign up at neon.tech (free, no card required).
2. Create a project → copy the connection string it gives you. It looks like:
   `postgresql://user:password@ep-xxxx.region.aws.neon.tech/dbname?sslmode=require`
3. Convert it to the asyncpg form the backend expects — replace `postgresql://` with `postgresql+asyncpg://`, and append `&ssl=require` if `sslmode=require` is already in the query string (asyncpg uses a different SSL param name). Final shape:
   `postgresql+asyncpg://user:password@ep-xxxx.region.aws.neon.tech/dbname?ssl=require`
4. Load the schema (tables, indexes, materialized views, the `refresh_materialized_views()` function) into Neon. From the project folder, using `psql` (or Neon's web SQL editor — paste the file contents there instead):
   ```bash
   psql "postgresql://user:password@ep-xxxx.region.aws.neon.tech/dbname?sslmode=require" -f database/schema.sql
   ```

Keep the `postgresql+asyncpg://...` connection string — you'll paste it into Render next.

## 3. Backend — Render

1. Sign up at render.com (free, GitHub login works).
2. Dashboard → **New** → **Blueprint** → pick your `indian-fin-platform` repo. Render reads `render.yaml` automatically and proposes one service, `indian-fin-backend`.
3. Before deploying, set the environment variables it flags as missing (`sync: false` in render.yaml):
   - `DATABASE_URL` → the Neon connection string from step 2
   - `FMP_API_KEY` / `ALPHA_VANTAGE_API_KEY` → your keys, if you have them (optional — only needed for those specific data providers)
4. Deploy. First build takes a few minutes. When done, Render gives you a public URL like `https://indian-fin-backend.onrender.com`.
5. Sanity check: open `https://indian-fin-backend.onrender.com/health` — should return `{"status":"ok",...}`.

Note: Render's free tier sleeps after 15 minutes idle and takes ~30-60s to wake on the next request. Fine for occasional/demo use; if you need it always-on later, that's a paid-tier upgrade.

## 4. Frontend — Streamlit Community Cloud

1. Sign up at share.streamlit.io with your GitHub account.
2. **New app** → select the `indian-fin-platform` repo, branch `main`, main file path `frontend/app.py`.
3. Under **Advanced settings → Secrets**, add:
   ```
   BACKEND_URL = "https://indian-fin-backend.onrender.com"
   ```
4. Deploy. You'll get a public URL like `https://<something>.streamlit.app` — that's the link to share.

## 5. Verify

Open the `.streamlit.app` URL from any device/browser. It should load the full UI and talk to the Render backend (give Render ~30-60s to wake up on the very first request after idle).

---

**Env vars reference**

| Service | Var | Source |
|---|---|---|
| Render (backend) | `DATABASE_URL` | Neon connection string (asyncpg form) |
| Render (backend) | `SECRET_KEY` | auto-generated by render.yaml |
| Render (backend) | `FMP_API_KEY`, `ALPHA_VANTAGE_API_KEY` | your own API keys (optional) |
| Streamlit Cloud (frontend) | `BACKEND_URL` | the Render backend's public URL |
